RandA commands library

RAComm is a library that allows to have a dialog between Arduino (RandA) and Raspberry. Begin function send a wake-up to waiting background process on Raspberry for starting communication by serial port.

This library allows to send command to Linux shell, open and use Raspberry files, and finally to open and use a Xterminal as console.

 

Functions:

 

void RAComm.begin()

Starts listen program on Raspberry.

 

bool RAComm.isReady()

Returns true if Raspberry ready to receive command.

 

bool RAComm.sendCommand(char *command)

Sends string command (program or script) to Raspberry and returns true if command executed correctly.

Es.: sendCommand("ls -l /home/pi").

 

char* RAComm.getAnswer(char repbuff[], int repbufflen)

Reads (into repbuff) a line of replay to command (if any), and returns NULL if no more lines. It reads until linefeed (\n) or max repbufflen-1 characters.

 

void RAComm.close()

Ends dialog. Raspberry listen command releases serial port and remains waiting next wake-up.

 

bool RAComm.openFile(char *name, char *mode,char *prot)

Opens file. Name is the path, mode is read,write append, prot is the octal code of protetion.

Es.: openFile("/home/pi/myfile.txt,"w","777")

 

bool RAComm.openFileRead(char *name)

bool RAComm.openFileWrite(char *name)

Specialized form.

 

char* RAComm.readRec(char recbuff[], int recbufflen)

Reads a line from opened file. If EOF it returns NULL.

 

bool RAComm.writeRec(char repbuff[])

Writes record on opened file.

 

bool RAComm.closeFile()

Closes opened file.

 

bool RAComm.openConsole()

Opens a Xterminal as console, for debugging or for send and read data interactivelly. (because serial port is used for Raspberry-Arduino communication)

 

char* RAComm.readConsole(char recbuff[], int recbufflen)

Reads a line from console. If no data it returns NULL.

 

bool RAComm.writeConsole(char repbuff[])

Writes record on console.

 

bool RAComm.closeConsole()

Closes console.

 

bool RAComm.setEcho()

Opens console and echoes every serial data for debugging.
